## Directory Purposes

**`frontend/app/`:**
- Purpose: Next.js App Router entry — pages, layouts, API routes.
- Contains: `page.tsx` (landing), `layout.tsx`, `not-found.tsx`, `globals.css`, `api/waitlist/route.ts`.
- Key files: `app/page.tsx` (the product), `app/api/waitlist/route.ts` (the backend boundary).

**`frontend/lib/`:**
- Purpose: Server-side integration clients (third-party SDK wrappers).
- Contains: `supabase.ts`, `resend.ts`.
- Key files: `lib/supabase.ts`, `lib/resend.ts` — both export a single lazy-init factory.

**`frontend/supabase/`:**
- Purpose: SQL migrations / schema for the waitlist feature.
- Contains: `waitlist.sql` (table + RLS). Run manually in Supabase SQL editor.

**`frontend/public/`:**
- Purpose: Static assets served as-is.
- Contains: Logo SVGs/PNGs, icons, favicon. `Elements/` is currently empty.

**`docs/`:**
- Purpose: Product/user documentation separate from code.
- Contains: `USER_FLOW.md`.

**Repo root (`/`) — non-code:**
- `ROADMAP.md`: canonical future-build plan (contracts/, services/, packages/ described here do NOT exist yet). Trust `frontend/` over `ROADMAP.md` when they conflict.
- `agents.md`: agent operating notes.
- `README.md`: intentionally empty.

## Naming Conventions

**Files:**
- Route files: `route.ts` inside an `app/api/<name>/` directory (Next App Router convention).
- Pages: `page.tsx`, `layout.tsx`, `not-found.tsx` (Next reserved filenames).
- Library modules: `lib/<service>.ts` (e.g., `supabase.ts`, `resend.ts`).
- React components: PascalCase (`Home`, `WaitlistForm`, `RootLayout`, `NotFound`).
- Config: kebab-case (`next.config.mjs`, `tailwind.config.ts`, `postcss.config.mjs`, `eslint.config.mjs`).
- SQL: `kebab-case.sql` (`waitlist.sql`).

**Directories:**
- `app/` (App Router), `lib/` (utilities/clients), `public/` (static), `supabase/` (migrations) — lowercase, singular.

**Functions:**
- Factory accessors named `get<Service>()` (`getSupabase`, `getResend`).
- Route handlers exported as uppercase HTTP verbs (`POST`).

## Where to Add New Code

**New API endpoint:**
- Create `frontend/app/api/<name>/route.ts` exporting `POST`/`GET`/etc.
- For server-only external clients, add a `getX()` factory in `frontend/lib/`.
- Never import `lib/` server clients into `app/**` client components (no `NEXT_PUBLIC_` secrets).

**New landing section / page:**
- Primary code: `frontend/app/page.tsx` (add a `<section>` + content array) or a new `frontend/app/<route>/page.tsx`.
- Styling: use Tailwind utilities + shared classes from `frontend/app/globals.css` / `frontend/tailwind.config.ts`.

**New data table / migration:**
- Add SQL to `frontend/supabase/` (e.g., `frontend/supabase/<feature>.sql`); run manually in Supabase SQL editor.

**New static asset:**
- Place in `frontend/public/` (e.g., `frontend/public/Icons/`, `frontend/public/Logo/`).

**Utilities:**
- Shared server helpers: `frontend/lib/`. Shared client helpers: co-locate in the component file or a new `frontend/lib/` module imported only by client code.

## Special Directories

**`frontend/.next/`:**
- Purpose: Next.js build/cache output.
- Generated: Yes (by `next build`/`next dev`).
- Committed: No (gitignored).

**`frontend/node_modules/`:**
- Purpose: Installed dependencies (pnpm).
- Generated: Yes.
- Committed: No.

**`frontend/public/Elements/`:**
- Purpose: Intended for visual SVG elements.
- Generated: No.
- Committed: Yes, but currently empty.

**`.planning/`:**
- Purpose: GSD planning docs (including this analysis).
- Generated: Yes (by GSD commands).
- Committed: Per repo policy (not part of the app).
