  <action>
    Define `pub trait Kms: Send + Sync { fn encrypt(&self, key_id: &str, plaintext: &[u8]) -> Result<Vec<u8>, CustodyError>; fn decrypt(&self, key_id: &str, ciphertext: &[u8]) -> Result<Vec<u8>, CustodyError>; fn sign(&self, key_id: &str, msg: &[u8]) -> Result<Vec<u8>, CustodyError>; }`.
    Implement `pub struct InMemoryKms { keys: Mutex<HashMap<String, Vec<u8>>> }` where `encrypt` stores `plaintext` under `key_id` (it is a stub — NOT real crypto, documented as test-only), `decrypt` returns it, `sign` performs an actual ed25519 sign using the stored seed via `ed25519-dalek`.
    Implement `pub fn from_config(cfg: &str) -> Result<Box<dyn Kms>, CustodyError>` parsing JSON like `{"provider":"memory"|"aws-kms"|"gcp-kms"|"vault", ...}`. Only `"memory"` is implemented now; other providers return `CustodyError::ProviderUnsupported` (so the crate compiles & tests run with no cloud creds). The real cloud providers are wired in a later phase; this plan MUST NOT require AWS/GCP/Vault SDKs to compile.
    Define `pub enum CustodyError` (thiserror): `Encrypt`, `Decrypt`, `Sign`, `KeyNotFound`, `ProviderUnsupported`, `Stellar`, `Supabase`, `InvalidKey`.
  </action>
  <acceptance_criteria>
    - `Kms` trait compiles and is object-safe (`Box<dyn Kms>`).
    - `InMemoryKms` round-trips encrypt→decrypt and produces a valid ed25519 signature.
    - `from_config("{\"provider\":\"memory\"}")` builds an instance; other providers error without crashing compile.
  </acceptance_criteria>
